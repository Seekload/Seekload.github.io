底层数据结构

Go 语言使用 eface 和 iface 分别描述空接口和非空接口，eface 代码如下：

1
2
3
4

type eface struct {    //16字节
	_type *_type
	data  unsafe.Pointer
}

由于空接口类型不包含任何方法，所以 eface 的结构比较简单，_type 指向底层数据类型，data 是指向原始数据的指针。

iface 代码如下：

1
2
3
4

type iface struct {    // 16个字节
	tab  *itab
	data unsafe.Pointer
}

iface 结构体包含两个成员，tab 指向 *itab 实体，data 是指向原始数据的指针。下面我们来详细介绍下 _type 和 itab。

_type 结构体

_type 是类型的运行时表示，包含了所有类型的元信息，比例：类型的大小、种类等。

type _type struct {
	size       uintptr // 类型占用内存大小
	ptrdata    uintptr

size 表示类型占用内存大小，单位是字节；kind 表示基础类型枚举值。其他字段的作用和含义大家可以自行了解。

接下来要说的 itab 结构体就包括一个 *interfacetype 成员。

itab 结构体

itab 结构体可以说是接口类型最为核心的部分，如下：

1
2
3
4
5
6
7

type itab struct {      // 32个字节
	inter *interfacetype
	_type *_type
	hash  uint32   // copy of _type.hash. Used for type switches.
	_     [4]byte
	fun   [1]uintptr
}

从上面的代码可以看出，itab 是接口静态类型与动态类型的组合，inter 表示接口的静态类型，_type 表示动态类型。itab.hash 与 _type.hash 是完全一致的，这个值在类型查询的时候会用到，后面我们再写文章分析。fun 字段是一个函数指针，指向动态类型的函数方法，实现接口调用方法的动态派发。是不是你也很好奇，为什么它的大小只有 1？如果我的动态类型定义了很多个方法怎么办？没关系，因为函数指针在内存中都是连续存放的，后面通过汇编我们可以看到，如果想要调用别的方法，通过移动地址就可以获取其他函数指针。

package main

import "fmt"

type Person interface {
	Speak()
	Play()
}

type Student struct {
	Name string
}

func (s Student) Speak() {
	fmt.Println(s.Name)
}

func (s Student) Play() {
	str := s.Name + " play."
	fmt.Println(str)
}

func main() {
	s := Student{Name: "seekload"}
	var p Person = s
	p.Speak()
}

上面的代码非常简单，自定义的结构体 Student 实现了 Person 接口。在主函数里面初始化结构体变量 s，并将 s 赋值给了接口变量 p，这个过程会发生类型转换，最后调用了 Spesk() 方法。

func convTstring(val string) (x unsafe.Pointer) {
	if val == "" {
		x = unsafe.Pointer(&zeroVal[0])
	} else {
		x = mallocgc(unsafe.Sizeof(val), stringType, true)
		*(*string)(x) = val
	}
	return
}

func InterfaceIsNil(i interface{}) {
	if i == nil {
		fmt.Println("nil interface")
		return
	}
	fmt.Println("non-nil interface")
}

func main() {
	var v interface{} = nil
	var i *int = nil

	InterfaceIsNil(v)
	InterfaceIsNil(i)
}

变量 i 明明是 nil，为什么经过 InterfaceIsNil() 输出 non-nil interface 呢？相信经过整篇文章的介绍，你已经知道原因是什么了。当调用 InterfaceIsNil() 入参是变量 i 时，其实在底层发生了类型转换，虽然此时动态值是 nil，但是动态类型 eface._type 却是 *int，不为 nil，所以此时 i != nil。
